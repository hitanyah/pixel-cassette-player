# 🧪 專案測試與驗證文件 (Testing & Verification Document)

本檔案說明的測試環境設定、測試用例 (Test Cases) 以及常見 CORS 問題與音訊問題排查。

---

## 1. 測試環境要求
* **作業系統**：Windows / macOS / Linux
* **瀏覽器**：Google Chrome, Mozilla Firefox, Safari（需啟用 HTML5 Canvas 與 Web Audio API 支援）
* **音訊裝置**：耳機或揚聲器（需確認系統音效正常輸出）
* **Spotify 要求**：非付費帳號亦可使用手動模式，如需串接 Spotify 歌單需準備一個 Spotify App 授權 Client ID。

---

## 3. 常見 CORS 與音訊錯誤排查

### CORS (跨來源資源共享) 錯誤
* **現象**：當音樂播放時，音樂正常播出，但是 Canvas 波形頻譜完全沒有動靜，且主機 Console 出現 `Access to audio at ... has been blocked by CORS policy`。
* **原因**：音訊來源伺服器（如某些第三方 MP3 網址）未返回 `Access-Control-Allow-Origin: *` 標頭，瀏覽器出於安全性限制，拒絕讓 Web Audio API 的 `MediaElementAudioSourceNode` 讀取其音訊緩衝區資料。
* **專案的應對設計**：
  * 對於 **Spotify 試聽 API**，其 CDN 預設支援 CORS，因此程式會動態呼叫 `audio.crossOrigin = 'anonymous'` 取得實體頻譜資料。
  * 對於其他 **Mock 免版權音樂**，若不支援 CORS，程式會自動移除 `crossOrigin` 屬性，此時**音樂依然可以正常播放，且我們設計的 Canvas 模擬演算法會自動接管**，產生極度逼真的隨機畫素頻譜，不影響使用者視覺體驗。

### 音訊內容無法自動播放 (Autoplay Blocked)
* **現象**：載入頁面並點選 Play 後，音樂沒有聲音，計時器被卡在 00:00。
* **原因**：現代瀏覽器安全性原則禁止網頁在使用者「與頁面進行首次互動」前自動播放聲音，並且在使用者點選前，音訊內容 `AudioContext` 會被強制設為 `suspended` (掛起) 狀態。
* **專案的應對設計**：
  * 我們在使用者點選 `PLAY` 按鈕、`FF`、`REW` 時，會明確且主動地呼叫 `audioContext.resume()`。由於此為使用者明確的點選互動 (Click Action)，瀏覽器會百分之百許可，並順暢開啟聲音管線。
